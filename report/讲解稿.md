# HW4 南开校内 Web 搜索引擎讲解稿

## 0:00-1:00 开场

大家好，我本次作业实现的是面向南开大学校内资源的 Web 搜索引擎。系统覆盖助教要求中的网页抓取、文本索引、查询服务、个性化查询、Web 界面和个性化推荐。需要强调的是，排序核心按原文要求使用向量空间模型，也就是 TF-IDF 余弦相似度，并融合 PageRank 链接分析结果。

## 1:00-2:30 系统架构

系统分为离线和在线两部分。离线部分负责抓取南开站点、解析 HTML 和附件、保存网页快照、构建本地多字段索引、批量写入 Elasticsearch、聚合锚文本并计算 PageRank。在线部分使用 Flask 提供搜索页面、登录注册、查询日志、搜索建议、网页快照和个性化排序。

这里 Elasticsearch 负责成熟的候选召回和过滤能力；如果 ES 没有启动，系统会自动回退到本地索引。候选结果进入同一套 VSM + PageRank 重排，因此排序模型仍然对应作业原文要求。

报告和 PPT 中已经放入系统首页、查询结果、网页快照、抓取分布和 selfcheck 的运行证据。

## 2:30-4:00 抓取与数据

爬虫只深入 `*.nankai.edu.cn`，遵守 `robots.txt`，按主机限速，并保存 HTML 快照。第一轮抓取完成 10 万页规模后，我又按高质量策略定向补爬 32052 条 HTML 页面；补爬前先执行 `pixi run targeted-crawl -- --dry-run`，把教务、招生、就业、图书馆、学院和科研站点的 frontier 按主机配额重排，并过滤 CAS 登录、跳转和噪声 URL。

当前真实语料已经达到 132296 条唯一文档，外链数量为 0，覆盖 356 个南开主机。其中 HTML 124844 条，PDF 3902 条，DOCX 1417 条，DOC 1411 条，XLSX 547 条，XLS 175 条。HTML 快照共 124844 份，附件或附件 URL 共 7452 条，索引文件约 1.98 GB。本轮定向扩展后 frontier 达到 384801 条，后续仍可在这个基础上继续增量抓取并重建索引。

## 4:00-5:30 索引与排序

索引字段包括标题、正文、URL、锚文本、附件文本、文档类型、快照路径和 PageRank。本地索引用 jieba 搜索模式和南开自定义词构建 TF-IDF 统计，ES 使用同样的字段做 `multi_match`、短语、通配和过滤召回。

排序部分采用 VSM：每个字段分别计算 TF-IDF 余弦相似度，再按字段权重融合。标题权重最高，锚文本其次，正文和附件文本也参与排序。最后再乘上 PageRank 增益，并加入时间新鲜度和个性化信号。

为了提升真实搜索质量，我还加入了多项重排后的优化。第一是结果去重和主机多样性：对标题和正文前缀计算内容签名，合并重复导航页，并限制首页结果中单个 host 的占比。第二是智能摘要：不再简单截取第一次命中，而是按句子和命中密度选择正文片段，跳过导航、版权和列表噪声。第三是质量修正：CAS 登录页、跳转页、URL 标题页和空正文页会降权；不可读 PDF 文件名会优先回退到报告、通知、办法等正文标题。第四是编码修复和主题意图：像 CNKI 页面里的乱码会自动修复，“就业”“选课”“图书馆”“人工智能”等查询会提升更符合校园主题的结果。

## 5:30-9:30 六项查询功能

第一，站内查询。示例是 `就业 实习 双选会 site:career.nankai.edu.cn`，结果只返回就业指导中心域名下的页面，并把 2026 年就业实习双选会、新闻报道和学院专场双选会排在前面。优化后的摘要会优先展示时间、地点和正文说明，而不是页面顶部菜单。

第二，文档查询。示例是 `国家级 实验教学示范中心 年度报告 filetype:pdf`，结果只返回 PDF，并显示文件类型、摘要命中和下载入口。

第三，短语查询。示例是 `"南开 大学"`，系统要求词项邻接，因此会区别于非邻接短语。

第四，通配查询。示例包括 `温*`、`计?` 和 `/信息.{0,2}/`。系统会将通配符转换为正则，并在标题、正文、锚文本和 URL 中匹配。

第五，查询日志。登录用户的搜索会写入 SQLite，首页显示最近查询历史。

第六，网页快照。搜索结果中的“快照”按钮会打开本地保存的 gzip HTML，并在顶部显示抓取时间和原始 URL。

项目目录移动后，旧绝对路径可能失效。系统在读取文档元数据时会自动把快照和附件路径重定位到当前 `data` 目录，因此压缩包解开或换目录后，快照和下载功能仍然可用。

## 9:30-11:30 个性化与推荐

系统支持注册和登录。注册时用户选择角色和兴趣标签。个性化排序中，本科生和研究生更偏 HTML 教务、课程和就业通知，教师更偏学术、图书馆和 PDF、DOCX、XLSX 等文档资源。结果页右侧还会展示带推荐理由的相关推荐，搜索框会通过 `/api/suggest` 提供联想。

演示时可以注册两个用户：一个选择本科生、教务、课程和就业，一个选择教师、学术和图书馆。对“人工智能 学术 讲座”查询，访客、本科生和教师画像的 Top1 会分别落到通用学术结果、就业实习相关结果和学术/校友新闻结果，相关推荐也会显示兴趣命中、同主题和文档资源等理由。

## 11:30-13:00 自检与运行方式

运行 `pixi run selfcheck`，可以看到索引、六项查询、VSM、ES 查询 DSL、个性化和 Web 路由全部 PASS。常用命令包括：

```powershell
pixi run init-demo
pixi run selfcheck
$env:NKU_SEARCH_ES_URL="http://127.0.0.1:9200"
pixi run build-es-index
pixi run search -- "国家级 实验教学示范中心 年度报告 filetype:pdf"
pixi run serve
pixi run targeted-crawl -- --dry-run
pixi run targeted-crawl -- --target-new 30000
.\.pixi\envs\default\python.exe .\scripts\crawl_to_100k.py
```
